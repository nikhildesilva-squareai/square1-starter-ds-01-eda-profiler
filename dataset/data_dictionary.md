# Data dictionary — `ecommerce_transactions.csv`

A raw export of marketplace orders (Jan 2024 – Dec 2025). **Synthetic — generated by Square 1 AI, free for learners to use.** ~50,500 rows.

> ⚠️ This is a *raw* export, not a clean dataset. Like real data, it contains
> quality issues you'll need to detect and handle. Part of the project is
> finding them — don't assume the columns are tidy.

| Column | Type | Description |
|---|---|---|
| `order_id` | string | Order identifier, e.g. `ORD-100918`. Not guaranteed unique. |
| `order_datetime` | string | Order date. **Formats are inconsistent** across rows. |
| `customer_id` | string | Customer identifier, e.g. `CUST-00590`. |
| `category` | string | Product category. Casing/spacing is **not normalised**. |
| `product` | string | Product name. |
| `unit_price` | float | Price per unit in AUD. |
| `quantity` | int | Units ordered. |
| `country` | string | 2-letter country code. **May be blank.** |
| `payment_method` | string | One of card / paypal / apple_pay / bank_transfer. **May be blank.** |
| `is_refunded` | string | `true` / `false`. |

**Known real-world characteristics to handle:** missing values, exact duplicate rows, mixed date formats, inconsistent category casing, and numeric outliers (including at least one impossible value). Your profiler should surface these automatically.
